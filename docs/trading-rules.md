# 매매 규칙 (Normative Spec)

이 문서는 coinAgent의 **리스크 엔진과 전략이 반드시 따라야 하는 규범**이다.
코드(리스크 게이트, 전략, 백테스트 비용 모델)는 이 문서를 기준으로 구현·테스트한다.

## 1. 자금·리스크 (하드 가드레일 — 코드로 강제)

| 규칙 | 값 | 강제 지점 |
|---|---|---|
| 레버리지 | 알트 3~5배, BTC 최대 10배 | RiskEngine이 주문 전 검증, 초과 시 거부 |
| 마진 모드 | 격리(Isolated) 고정 | 브로커가 진입 전 항상 isolated 설정 |
| 주문 유형 | 지정가(limit)만 | Broker 인터페이스에 market 주문 없음. 수수료 maker 0.025% / taker 0.05% 가정 |
| 몰빵 금지 | 분할 진입·분할 청산 (기본 50% / 25% / 25%) | 시나리오(TradePlan)에 분할 레그 필수 |
| 손익비 게이트 | 알트 ≥ 1:3, BTC·ETH ≥ 1:2 | 손익비 미달 TradePlan은 진입 거부 |
| 복리 금지 | 수익은 출금 원장(withdrawal ledger)으로 분리, 시드 고정 | 사이클 종료 시 시드 초과분 자동 '출금' 처리 |
| 저시총 금지 | 화이트리스트(BTC·ETH·SOL·XRP·DOGE 등 설정값) 외 심볼 매매 금지 | RiskEngine 검증 |

## 2. 진입 조건

- **근거 2개 이상**: 지지선, 이평선 리테스트, RSI 과매도, VWMA 지지 등 독립 근거가 2개 미만이면 관망. 전략 시그널은 근거 목록(evidence[])을 반환해야 하며 RiskEngine이 개수 검증.
- **시나리오 필수**: 진입 전 TradePlan(근거 + 손절선 + 분할 익절선 + 분할 진입 비중)이 완성돼야 주문 가능. 시나리오 없는 주문은 브로커가 거부.
- **지표 발표 블랙아웃**: CPI·FOMC 등 이벤트 전후(기본 ±12시간, 설정 가능)는 신규 진입 금지. 이벤트 캘린더는 설정/DB로 관리.
- **홀짝 자리 금지**: 채널·박스 중간(손익비 ≈ 1:1) 위치에서는 진입 금지 — 손익비 게이트가 자연 차단하지만, 박스 전략은 박스 상·하단 근접(기본 하위/상위 25% 구간)에서만 시그널 생성.
- **저시총 금지**: 화이트리스트 외 심볼은 롱/숏 불문 금지.

## 3. 손절·익절

- **손절선 = 시나리오 붕괴 지점** (매물대·채널 하단 이탈). 이탈 판정은 **4시간봉 마감 기준** — 장중 위크(꼬리)로는 손절하지 않고, 4h 종가가 손절선을 이탈하면 청산.
- **분할 익절**: 1차 저항(예: VWMA 100)에서 50%, 2차 저항(200선)에서 잔량 전량 — TradePlan의 take-profit 레그로 표현.
- **박스권**: 반대편 끝까지 기다리지 않고 중간(박스 미드포인트)에서 익절.
- **계획 라인 무조건 준수**: 손절 미루기 금지 — 손절/익절 라인 수정은 '유리한 방향'(손절 타이트닝, 익절 보수화)만 허용. 손절선을 불리하게 넓히는 수정은 코드가 거부.

## 4. 매매 방법 (전략 카탈로그)

1. **톱다운 타점 매매 (기본기)** — 일봉 200선 위 확인 → 4h 골든크로스(50↑200)+눌림 → 15m/5m 지지+거래량 확인 → (1m 정밀 손절선) 진입. "큰 흐름 읽고 작게 들어간다."
2. **이평선 황금타점** — 50·100·200선이 한 지점에 수렴하는 자리. 지지 시 눌림목 매수, 이탈 시 칼손절. 400선은 패닉셀 급락 시 분할 매수 기준.
3. **TOTAL2/3 + 도미넌스 시장 판단** — TOTAL2·3 50선이 200선 위면 롱장, 아래면 숏/관망. 시장↑+도미넌스↓ = 알트 불장, 시장↓+도미넌스↑ = 현금/숏, 둘 다 하락 = 시장 이탈. (레짐 필터로 모든 전략에 적용)
4. **상대강도 종목 선정** — 유니버스 동시 비교. BTC 하락 시 덜 빠지는 종목 = 롱 후보, 더 빠지는 종목 = 숏 후보.
5. **매물대·추세선·채널 매매** — 매물대는 4h·일봉에서 꼬리 포함 '구간'으로, 리테스트로 검증. 박스: 지지=롱, 저항=숏, 중간=익절. 추세선은 저점 2개(상승)/고점 2개(하락). 추세선=진입 타이밍, 매물대=손절·익절 기준. 거래량 없는 돌파는 페이크아웃 의심.
6. **VWMA·VPVR 보조 판단** — VWMA 100 위 지지 = 상승 유효, 조정 시 그 선까지 대기 후 매수. VPVR 매물대가 횡보 구간에 쌓이면 매집 → 재상승 기대. VWMA+VPVR 동시 하향 이탈 = 매도/숏.
7. **캔들 패턴 대응** — 장대양봉+거래량 = 돌파 매매 가능(눌림 확인 후) / 저항대 윗꼬리 음봉 = 숏 고려 / 지지대 아래꼬리 양봉 = 롱 고려 / 도지 = 다음 캔들 확인.
8. **순환매 (숙련자용, 물렸을 때)** — 유의미한 구간(전고점·200선)에서 추가 진입해 평단 조정 → 되돌림에 일부 정리 → 평단 이상 전량 청산. 강한 반대 추세면 즉시 손절(순환매 금지), 추가 진입 후에도 반대로 가면 추가분만 빠른 손절. 수수료 포함 실제 BEP 기준 계산. (기본 비활성, 설정으로만 켬)
9. **회피 규칙 (세력·호가창)** — 계단식 상승, 끊기는 체결, 매집봉(역망치+윗꼬리+거래량 폭발 후 무거래 음봉) 감지 시 접근 금지. 저점의 두꺼운 매도벽 = 허매도벽(매수 기회) 가능, 지지 붕괴 직전 두꺼운 매수벽 = 허매수벽(덤핑 전조) 의심.

## 5. 구현 매핑 노트

- 규칙 1·2·3은 **RiskEngine**(주문 전 게이트) + **TradePlan**(시나리오 자료구조)으로 구현하며, 전략이 어기려 해도 통과 불가.
- 전략 1~7은 백테스트 가능한 결정론적 Strategy 클래스로 구현. 8·9는 실시간 보조 판단(기본 off / 로그 경고)으로 구현.
- 백테스트 비용 모델: maker 0.025%/side + 슬리피지 + 펀딩비 근사. 지정가 미체결 리스크는 fill 시뮬레이션(봉이 지정가를 관통해야 체결)으로 반영.
